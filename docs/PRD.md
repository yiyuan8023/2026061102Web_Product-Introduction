# 产品工具合集官网 PRD

## 1. 背景

当前已有“定时任务调度器”桌面产品，后续还会持续出现多个同类型的小产品。需要一个客户可访问的产品合集官网，用于统一展示产品入口，并为每个产品提供独立使用说明。

站点已绑定域名 `data-365.cn`，备案号为 `浙ICP备2023041540号-3`。

## 2. 目标

- 让客户快速知道当前有哪些可用产品。
- 让客户点击产品后进入独立说明页，通过截图和步骤理解用途与操作方式。
- 让后续新增小产品时不需要重做官网结构。
- 保持官网表达克制，重点是产品集合与使用说明，不做单产品强营销。
- 让维护人员可以通过 GitHub 管理源码，并让服务器从 GitHub 拉取更新。

## 3. 目标用户

主要用户：

- 客户：查看产品用途、场景、界面和基础操作。
- 交付人员：向客户发送产品说明链接。
- 维护人员：新增产品、更新截图、调整文案和上线静态页面。

客户关心的问题：

- 这个产品解决什么问题。
- 适合什么场景。
- 如何开始使用。
- 界面长什么样。
- 遇到常见问题怎么处理。
- 后续还有哪些工具会继续扩展。

## 4. 页面范围

### 4.1 首页

首页作为产品合集入口，包含：

- 顶部导航
- 产品合集首屏
- 产品卡片列表
- 产品分类和筛选
- 适用场景说明
- 新增产品扩展说明
- 备案信息

首页不做单产品落地页，不把某一个产品包装成强营销首屏。

### 4.2 产品详情页

每个产品拥有独立页面，包含：

- 产品名称和一句话说明
- 真实界面配图
- 适用场景
- 核心能力
- 使用步骤
- 界面模块说明
- 常见问题
- 返回产品合集入口
- 备案信息

## 5. 当前首个产品

### 定时任务调度器

一款本地桌面调度工具，用于管理和执行 Python 脚本、内置任务和自动化作业。

核心展示点：

- 支持 Cron、固定间隔、指定时间三类触发方式。
- 支持本地脚本调度，适合数据同步、爬虫采集、定时报表和自动化任务。
- 支持运行日志、告警记录和日志查询。
- 支持系统托盘常驻，关闭窗口后任务仍可继续运行。
- 使用 SQLite 保存任务配置，重启后可恢复。

## 7. 内容标准

- 文案优先讲用途、场景、入口、动作、结果和异常处理。
- 产品详情页必须优先使用真实软件截图。
- 不在客户页面暴露过多内部实现、临时路径、账号密码或服务器细节。
- 后续产品未完成时可以在首页保留规划中卡片，但不要链接到空白详情页。
- 备案号必须在首页和详情页底部展示，并链接到工信部备案系统。

## 8. 视觉标准

- 客户侧官网要清爽、可信、偏工具型。
- 避免过度营销、过度装饰和单一蓝紫色调。
- 卡片用于单个产品项，不把整页做成一堆浮动卡片。
- 移动端优先保证文字不重叠、按钮不挤压、图片可完整查看。
- 新增截图时要保证文件体积适中、内容清晰、路径稳定。

## 9. 技术和部署要求

技术要求：

- 纯静态 HTML/CSS/JavaScript。
- 不引入构建工具，除非后续产品规模明确需要。
- 不依赖远程 CDN。
- 图片、CSS、JS 使用相对路径。

部署要求：

- 源码托管在 GitHub：`git@github.com:yiyuan8023/2026061102Web_Product-Introduction.git`。
- 服务器目录：`/var/www/data-365.cn`。
- Nginx 配置：`/etc/nginx/sites-available/data-365`。
- 服务器通过 GitHub 拉取代码更新。
- HTTPS 使用 Certbot 证书，证书由服务器自动续期。
- 不提交服务器密钥、Token、证书、临时包和本地运行目录。

## 10. 维护流程

本地维护流程：

1. 修改页面、样式或图片。
2. 运行 `python -m http.server 5178` 本地预览。
3. 检查首页、产品详情页、图片和备案号。
4. 提交 Git 并推送到 GitHub。
5. 登录服务器执行 `git pull --ff-only`。
6. 验证线上 HTTPS 访问。

服务器更新流程：

```bash
cd /var/www/data-365.cn
git pull --ff-only
sudo nginx -t
sudo systemctl reload nginx
```

## 11. 验收标准

- 首页可展示产品合集，并且不把单一产品放得过重。
- 当前产品有独立详情页。
- 详情页包含真实截图和清晰使用步骤。
- 首页和详情页均显示 `浙ICP备2023041540号-3`。
- 后续新增产品时，只需复制详情页模板并添加首页卡片。
- 本地 `python -m http.server 5178` 可正常预览。
- `https://data-365.cn/` 返回 `200 OK`。
- `https://www.data-365.cn/` 返回 `200 OK`。
- `http://data-365.cn/` 跳转到 HTTPS。
